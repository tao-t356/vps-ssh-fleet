# SECURITY

## 不要提交这些内容

- 私钥文件
- 公钥文件
- 真实服务器 IP / 域名清单
- `inventory/hosts.csv` 这类临时密码清单
- 任何密码、令牌、云厂商密钥

## 建议做法

1. 使用你自己生成的 SSH 密钥
2. 给私钥设置 passphrase
3. 为不同用途做密钥分离（例如办公 / CI / 临时环境）
4. 首次迁移完成后删除旧公钥
5. 只在确认公钥登录可用后关闭密码登录
6. 如果你临时把 VPS 密码写进了 `inventory/hosts.csv`，迁移完成后尽快删除或清空

## 如果曾使用过别人生成的密钥

请视为该密钥已经泄露，建议：

1. 新生成一对密钥
2. 批量部署新公钥
3. 删除所有 VPS 上的旧公钥
4. 重载 SSH 配置
